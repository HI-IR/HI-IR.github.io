ROOM框架

Room 是 Android 官方提供的一个用于处理 SQLite 数据库的持久化库，它在 SQLite 基础上进行了封装，简化了数据库操作，提供了更直观、高效且类型安全的数据库访问方式

主要组件

数据库（Database）

  • 这是 Room 框架的核心类，需继承 RoomDatabase 抽象类。
  • 利用 @Database 注解对其进行标注，该注解用于指定数据库包含的实体类和版本号。
  • 为每个 DAO（数据访问对象）提供一个抽象方法，返回对应的 DAO 实例

实体（Entity）

  • 代表数据库中的一张表，是一个 Java 或 Kotlin 类。
  • 用 @Entity 注解标注，可指定表名等信息。
  • 类中的每个字段对应表中的一列，可使用 @PrimaryKey 注解指定主键。

数据访问对象（DAO）

  • 是一个接口或抽象类，借助注解定义数据库操作方法，如插入、查询、更新和删除等。
  • 常见注解有 @Insert、@Query、@Update 和 @Delete。

工作流程

  1. 导入配置文件
  2. 创建实体类Entity
  3. 创建DAO接口
  4. 创建数据库类Database
  5. 封装数据库与业务逻辑交互的Repository

创建实体类Entity

注解（常用）

序号 注解名称 描述
1 @Entity 声明所标记的类是一个数据表，@Entity 包括的参数有：tableName(表名），indices（表的索引），primaryKeys（主键），foreignKeys（外键），ignoredColumns（忽略实体中的属性，不作为数据表中的字段），inheritSuperIndices（是否集成父类的索引，默认 false）
2 @ColumnInfo 用来声明数据库中的字段名，参数：name(在数据库中的名字)
3 @PrimaryKey 被修饰的属性作为数据表的主键，@PrimaryKey 包含一个参数：autoGenerate(是否允许自动创建，默认false)
4 @Embedded 把一个实体类的字段嵌入到另一个实体类当中,有时一个实体可能包含多个相关联的子字段，而这些子字段可以归为一个独立的逻辑单元。通过使用 @Embedded 注解，能将这些子字段封装成一个单独的类，再嵌入到主实体类中，从而让代码结构更清晰，更易于维护
5 @Dao 将类标记为数据访问对象（Data Access Object）
6 @Database 将类标记为RoomDatabase
7 @Delete 将 DAO 中的方法标记为与删除相关的方法
8 @Insert
9 @Update
10 @Query
